Kuriyr

The self-hosted notification microservice for SaaS developers.

Send transactional emails with React Email templates, built-in i18n, and a monitoring dashboard. One docker compose up and you're live.

Get Started View on GitHub

Send a translated email in one API call.

curl -X POST http://localhost:4400/send \
  -H "Content-Type: application/json" \
  -d '{
    "template": "welcome",
    "locale": "fr",
    "to": "user@example.com",
    "variables": { "name": "Jane" }
  }'

Features

Everything you need, nothing you don't.

React Email Templates

Write your email templates as React components. Preview them, reuse them, version them with your code.

Built-in i18n

Drop JSON locale files next to your templates. Kuriyr resolves translations and interpolates variables automatically.

Monitoring Dashboard

See every email sent, its status, and a full HTML preview. All from a simple self-hosted UI.

How it works

Three steps. That's it.

01

Write your templates

Create React Email components in the /templates folder with JSON locale files.

02

Configure & launch

Set your SMTP credentials or Resend API key via environment variables or kuriyr.config.ts, then run docker compose up.

03

Send via API

Call POST /send with a template name, locale, and variables. Kuriyr handles the rest.

Quick Start

Up and running in 60 seconds.

# Create a docker-compose.yml with your SMTP config
# (see the README for the full example)

docker pull ghcr.io/visionlab-tools/kuriyr:latest
docker compose up

Configure entirely via environment variables. Then open http://localhost:4400 to see the dashboard.
